## kafka topic setup
Setting up Kafka topic docker container.

issue: conflict over localhost when trying to ingress the Kafka topic server.

In `server.properites` ensure that docker container name (i.e. `kafka-topic-server`) is added to advertised as so:

```ini
advertised.listeners=PLAINTEXT://kafka-topic-server:9092
```

* `PLAINTEXT` is the protocol for sending messages in an unencrypted fashion
* `kafka-topic-server` the docker-host name
* `9092` the port kafka-server is bound to

On the host machine, for this to work ensure that kafka-topic-server is bound to the host like for example

```sh
# /etc/hosts
127.0.0.1 kafka-topic server
```

## Testing
On the host machine run this to produce

```sh
kafka/kafka/bin/kafka-console-producer.sh --bootstrap-server 0.0.0.0:9092 --topic test
```

... and run this on host again to consume ...

```sh
usr/local/kafka/bin/kafka-console-consumer.sh  --topic test --from-beginning --bootstrap-server localhost:9092
```

## Remarks

Note this arrangement allows crosss communication to consumers on docker and the native host.  Therefore, in a fully containerised setup, list the advertised listeners to be the broker/s associated with brokers - setting the /etc/hosts will not be necessary as long as the network is the same across the nodes.  Kubenetes may offer more with its own DNS configuration options to.
